# Журнал изменений

## 0.4.0 - 2025-12-23
- Добавлен слой Objects (`objects/index.json` + паспорта + селекторы наборов данных).
- Добавлен пример пакета objects: `examples/objects-two-objects`.

## 0.3.0 - 2025-12-23
- Добавлены Volume Sets: многофайловая поставка через `sitepack.volumes.json`, рекомендуемый размер частей — 100 MiB.
- Добавлены Chunked Assets: крупные blob-данные разбиваются на чанки и проверяются по `sha256`.
- Сохранена единая политика версий для всего проекта: package + envelope + volume-set.

## 0.2.0 - 2025-12-22
- Стандартизированы relation-ссылки (`Link`) и кодирование `relations: relationKey -> Link[]`.
- Добавлены рекомендуемые шаблоны ключей связей: `property.<CODE>`, `field.<CODE>`, URN-ссылки.
- Ужесточена схема сущностей для обязательного формата `relations`.
- Добавлен канонический пример `examples/cross-relations`: связи entity-to-entity и entity-to-asset через asset-index и blob.
- Синхронизированы схемы эталонных валидаторов и покрытие примерами.
- Версия заголовка/схемы `sitepack-envelope` приведена к 0.2.0 без изменений формата.

## 0.1.1 - 2025-12-22
- Стандартизированы relation-ссылки (`Link`) и массивное кодирование связей.
- Обновлена схема entity для проверки структуры связей.
- Расширен реестр сущностей рекомендуемыми шаблонами ключей связей.
